# 02-03 持久化导入任务与里程碑收尾

## Objective

用持久化、并发安全、可恢复的 job 模型替换进程内导入状态，并完成 v0.2 的最终质量门槛。

数据库迁移基础、reader 路由、响应最小化、CI 和依赖修复已验证，不在本计划重复实现。

## Implementation Slice 1: Persistent Import Job

### Steps

1. 新增 `import_jobs` ORM、状态枚举和 Alembic migration，关联 owner 与 novel。
2. 上传入口先创建唯一 job ID，并返回可轮询的持久状态。
3. 将 queued/running/succeeded/failed/cancelled 状态、进度、错误、重试次数和时间戳写入数据库。
4. 用租约或等价并发控制保证单 job 只有一个 worker 执行。
5. 增加幂等键、失败重试、取消和服务重启恢复测试。
6. **Test, Fix, and Confirm**：运行并发、重启、失败补偿和完整后端测试，直到无重复章节、孤儿文件或悬挂任务。

### Must-Haves

- 不再以进程内字典或负数临时 ID 作为事实源。
- job 状态跨进程重启保留，用户只能读取自己的 job。
- 重试不会重复创建小说、章节或后续索引。
- 文件、job、novel 和 chapter 的失败路径有明确事务/补偿边界。

### Verification

```powershell
cd backend
.\.venv311\Scripts\python.exe -m pytest -q -k "import_job or import_status or retry or recovery"
.\.venv311\Scripts\python.exe -m pytest -q
.\.venv311\Scripts\python.exe -m alembic upgrade head
.\.venv311\Scripts\python.exe -m alembic check
```

## Implementation Slice 2: Milestone Gate

### Steps

1. 运行后端测试、Ruff、Bandit、pip-audit 和 PostgreSQL migration 检查。
2. 运行前端 Vitest、typecheck、ESLint、production build 和 npm audit。
3. 执行认证、上传、阅读和 import job 的运行态 smoke。
4. 更新 `IMPLEMENTATION-STATUS.md`、`docs/` 和 `.gsd/`，只记录有证据的状态。
5. **Test, Fix, and Confirm**：修复所有回归并重复完整门槛，确认 v0.2 可关闭后再激活 RAG milestone。

### Must-Haves

- 所有自动检查无未接受的高风险结果。
- 文档与代码、migration head、测试数量和 active plan 一致。
- `/gsd auto` 在完成前保持 `02-03`，完成后才推进新 milestone。
