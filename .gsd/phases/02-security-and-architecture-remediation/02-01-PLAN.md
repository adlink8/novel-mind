# 02-01 仓库与上传边界修复

## Objective

先消除最容易造成密钥泄露、仓库污染和任意路径写入的输入边界风险。

## Implementation Slice 1: Git Ignore And Secret Boundary

### Steps

1. 把 `.gitignore` 行尾说明改为独立注释，覆盖 `.env`、虚拟环境、依赖、构建、缓存和上传目录。
2. 使用 `git check-ignore -v` 验证关键路径。
3. 检查已跟踪文件和暂存区，不读取或输出真实 secret 内容。
4. **Test, Fix, and Confirm**：重复执行 ignore/secret 检查，直到敏感和生成文件均不可进入普通提交。

### Must-Haves

- `backend/.env`、`frontend/node_modules/`、`frontend/.next/`、`__pycache__/`、`backend/uploads/` 被正确忽略。
- 不删除用户工作文件；仅修正规则并报告已跟踪风险。
- secret scan 不在日志中打印密钥值。

### Verification

```powershell
git check-ignore -v backend/.env frontend/node_modules frontend/.next backend/app/__pycache__ backend/uploads/test.txt
git status --short
git grep -n -I -E "(sk-[A-Za-z0-9_-]{16,}|BEGIN (RSA|OPENSSH|EC) PRIVATE KEY)" -- . ':!backend/.env'
```

## Implementation Slice 2: Upload Path And File Lifecycle

### Steps

1. 保存前丢弃客户端路径，仅保留显示用原始文件名。
2. 使用 UUID/随机存储名，并验证 resolved path 位于上传根目录。
3. 阻止同名覆盖，补偿数据库提交失败和删除失败。
4. 增加路径穿越、绝对路径、重复文件名和失败回滚测试。
5. **Test, Fix, and Confirm**：运行聚焦测试和完整后端测试，确认文件始终留在上传根目录。

### Must-Haves

- `../`、绝对路径、Windows drive/UNC 名称不能越界。
- 数据库失败不留下孤儿文件，删除失败不产生悬挂记录。
- 最大上传限制在读取过程中生效，不先把无限请求读入内存。

### Verification

```powershell
cd backend
.\.venv\Scripts\python.exe -m pytest -q tests/test_novels.py tests/test_services.py
.\.venv\Scripts\python.exe -m pytest -q
```
