# 02-02 认证、SSRF 与密钥保护

## Objective

为小说和 AI 模型配置建立身份边界，关闭任意服务端出站访问和明文密钥风险。

## Implementation Slice 1: Authentication And Authorization

### Steps

1. 明确本地单用户与多用户部署模式，定义 authenticated principal。
2. 给 novels、models 和生成类路由增加统一认证依赖。
3. 为资源增加 owner 关系并校验读写删除权限。
4. 增加未认证、跨用户访问和合法所有者测试。
5. **Test, Fix, and Confirm**：运行权限矩阵测试，确认所有非 health 路由采用一致策略。

### Must-Haves

- 未认证访问默认拒绝。
- 用户不能读取、修改或删除其他用户的小说和模型配置。
- health/readiness 例外必须显式列出。

### Verification

```powershell
cd backend
.\.venv\Scripts\python.exe -m pytest -q -k "auth or authorization or ownership"
.\.venv\Scripts\python.exe -m pytest -q
```

## Implementation Slice 2: SSRF And Error Disclosure

### Steps

1. 限制 provider URL scheme、host 和 port。
2. 解析 DNS 后阻断 loopback、private、link-local、multicast 和云 metadata 地址。
3. 对重定向执行相同校验，并配置出站 timeout/size 限制。
4. 对外返回稳定错误码和 correlation ID，详细异常只写脱敏日志。
5. **Test, Fix, and Confirm**：覆盖 IPv4、IPv6、DNS 重绑定和重定向案例。

### Must-Haves

- `localhost`、`127.0.0.0/8`、RFC1918、`169.254.0.0/16`、`::1`、ULA 被拒绝。
- 错误响应不包含内部 URL、堆栈、凭据或 provider 原始响应。

### Verification

```powershell
cd backend
.\.venv\Scripts\python.exe -m pytest -q -k "ssrf or model_url or error_redaction"
```

## Implementation Slice 3: Secret Storage

### Steps

1. 引入 envelope encryption 或操作系统/密钥管理服务支持的加密层。
2. 迁移已有明文 key，API 只返回掩码和指纹。
3. 删除数据库 URL 明文/截断日志，支持 key rotation。
4. **Test, Fix, and Confirm**：验证数据库和日志中不出现测试 key 明文。

### Must-Haves

- 主密钥不存入数据库或仓库。
- 读取模型列表永不返回 provider key。
- rotation 和旧数据迁移有回滚策略。

### Verification

```powershell
cd backend
.\.venv\Scripts\python.exe -m pytest -q -k "secret or encryption or redaction"
```
