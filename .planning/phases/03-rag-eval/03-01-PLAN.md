## Implementation Slice 1: 评测数据结构 + ORM

### Modules Owned
- `backend/app/models/eval.py` (NEW)
- `backend/app/schemas/eval.py` (NEW)

### Steps

1. 新增 `EvalDataset` ORM 模型：
   - 字段：`id, novel_id (FK), question, question_type, difficulty, gold_chunks(JSON), expected_points(JSON), must_not_say(JSON), status(candidate/confirmed/rejected), created_by`
   - Alembic migration
2. 新增 `EvalRun` ORM 模型：
   - 字段：`id, run_name, strategy(枚举), novel_id, total_questions, recall_k, precision_k, faithfulness_score, latency_ms, cost_usd, created_at`
3. 新增 `EvalResult` ORM 模型：
   - 字段：`id, run_id (FK), dataset_id (FK), recalled_chunks(JSON), answer_text, score, metrics(JSON), is_error_case`
4. 新增 Pydantic schema：`EvalDatasetCreate/Response`、`EvalRunCreate/Response`、`EvalResultResponse`、`EvalReportResponse`
5. 新增 `backend/evals/` 目录 + 示例 JSON 数据集
6. **Test, Fix, and Confirm**：Alembic upgrade/check、模型 test

### Must-Haves
- gold_chunks 存储为 JSON 数组（chunk ID 字符串）
- expected_points 和 must_not_say 为 JSON 字符串数组
- status 枚举：candidate / confirmed / rejected
- 每条测试题关联 novel_id

## Implementation Slice 2: 候选测试题生成器

### Modules Owned
- `backend/scripts/generate_eval_candidates.py` (NEW)

### Steps

1. 从 `text_chunks` 表中读取已索引的 chunk 内容
2. 对每部小说，按 5 种类型生成候选测试题：
   - 原文定位（20条）：从 chunk 内容直接改写为问句
   - 人物关系（20条）：跨 2-3 个 chunk 的关系型问题
   - 事件因果（20条）：跨章节的因果链问题
   - 时间线（20条）：事件顺序和时序问题
   - 伏笔/回收（20条）：长距离关联问题
3. 每条候选包含：question、question_type、gold_chunks（AI 推断）、expected_points（AI 生成）
4. 输出到 `evals/novel_eval_candidates.json`，全部标记 `status: candidate`
5. 生成后打印统计：总候选数、各类型分布
6. **Test, Fix, and Confirm**：对龙族 I 执行生成，验证各类型均有产出

### Must-Haves
- 使用现有 ai_service.chat() 生成，不引入新 LLM 调用方式
- gold_chunks 必须引用真实的 chunk_id（从 text_chunks 表查询）
- 每部小说至少生成 20 条候选（不足 100 条取 max）

### Verification
```bash
cd backend
python scripts/generate_eval_candidates.py --novel-id 1 --output evals/novel_eval_candidates.json
# 检查输出文件：每种类型 >= 1 条，gold_chunks 为有效 ID
```

## Implementation Slice 3: 自动评测引擎

### Modules Owned
- `backend/app/services/eval_service.py` (NEW)
- `backend/scripts/run_rag_eval.py` (NEW)

### Steps

1. 实现 `EvalService`：
   - `run_eval(run_name, strategy, novel_id, dataset_ids)`: 执行评测
   - 支持 3 种方案：`baseline_vector`（纯向量）、`hybrid_search`（BM25+向量）、`hybrid_worker`（留接口）
   - 对每条测试题，执行检索 → 对比 gold_chunks → 计算指标
   - 指标：recall@k、precision@k、MRR、NDCG@k
2. 实现 `run_rag_eval.py`：
   - 支持 `--strategy` / `--novel-id` / `--output` 参数
   - 输出：JSON 结果 + Markdown 报告
   - 错误案例导出（recall=0 的条目）
3. 新增 API 端点 `POST /api/eval/runs`（触发评测）、`GET /api/eval/runs/{id}/report`
4. **Test, Fix, and Confirm**：对龙族I 执行 baseline_vector vs hybrid_search 对比，输出报告

### Must-Haves
- 评测引擎不阻塞 HTTP 请求（异步执行）
- 至少支持 baseline_vector 和 hybrid_search 两种策略
- 每条测试题的检索结果完整记录（可追溯）
- recall@k 默认 k=5

### Verification
```bash
cd backend
python scripts/run_rag_eval.py --strategy hybrid_search --novel-id 1 --output evals/results/
# 检查输出：JSON 结果非空，报告可读
pytest tests/test_eval_service.py -v
```
