## 核心数据模型

```sql
-- 小说
CREATE TABLE novels (
    id UUID PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    author VARCHAR(255),
    description TEXT,
    genre VARCHAR(100),
    word_count INTEGER DEFAULT 0,
    chapter_count INTEGER DEFAULT 0,
    status VARCHAR(50) DEFAULT 'importing',  -- importing/ready/analyzing/analyzed
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 章节
CREATE TABLE chapters (
    id UUID PRIMARY KEY,
    novel_id UUID REFERENCES novels(id),
    chapter_number INTEGER,
    title VARCHAR(500),
    content TEXT NOT NULL,
    summary TEXT,
    word_count INTEGER DEFAULT 0,
    embedding VECTOR(1536),  -- pgvector 向量
    created_at TIMESTAMP DEFAULT NOW()
);

-- 人物
CREATE TABLE characters (
    id UUID PRIMARY KEY,
    novel_id UUID REFERENCES novels(id),
    name VARCHAR(255) NOT NULL,
    aliases JSONB,           -- 别名列表
    description TEXT,
    personality JSONB,       -- 性格特征
    role VARCHAR(50),        -- protagonist/antagonist/supporting/minor
    first_appearance INTEGER, -- 首次出场章节
    stats JSONB,             -- 出场统计
    created_at TIMESTAMP DEFAULT NOW()
);

-- 人物关系
CREATE TABLE character_relations (
    id UUID PRIMARY KEY,
    novel_id UUID REFERENCES novels(id),
    source_character_id UUID REFERENCES characters(id),
    target_character_id UUID REFERENCES characters(id),
    relation_type VARCHAR(100),  -- family/friend/enemy/mentor/lover/...
    description TEXT,
    strength FLOAT DEFAULT 0.5,  -- 关系强度 0-1
    chapter_range JSONB,         -- 关系存在的章节范围
    created_at TIMESTAMP DEFAULT NOW()
);

-- 时间线事件
CREATE TABLE timeline_events (
    id UUID PRIMARY KEY,
    novel_id UUID REFERENCES novels(id),
    event_title VARCHAR(500) NOT NULL,
    description TEXT,
    event_time VARCHAR(255),     -- 小说内的时间描述
    chapter_id UUID REFERENCES chapters(id),
    event_order FLOAT,           -- 排序用
    event_type VARCHAR(100),     -- battle/meeting/discovery/...
    importance VARCHAR(20),      -- high/medium/low
    characters_involved UUID[],  -- 涉及的人物 ID
    created_at TIMESTAMP DEFAULT NOW()
);

-- 同人文
CREATE TABLE fanfictions (
    id UUID PRIMARY KEY,
    novel_id UUID REFERENCES novels(id),
    title VARCHAR(500) NOT NULL,
    description TEXT,
    style_config JSONB,     -- 风格参数
    branch_point JSONB,     -- 从哪个时间点分支
    status VARCHAR(50) DEFAULT 'draft',
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 同人文章节
CREATE TABLE fanfiction_chapters (
    id UUID PRIMARY KEY,
    fanfiction_id UUID REFERENCES fanfictions(id),
    chapter_number INTEGER,
    title VARCHAR(500),
    content TEXT,
    ai_generated BOOLEAN DEFAULT FALSE,
    model_used VARCHAR(100),
    style_score FLOAT,       -- 风格一致性评分
    created_at TIMESTAMP DEFAULT NOW()
);

-- AI 模型配置
CREATE TABLE ai_configs (
    id UUID PRIMARY KEY,
    user_id VARCHAR(255),
    provider VARCHAR(50),    -- openai/anthropic/ollama/custom
    model_name VARCHAR(100),
    api_key_encrypted TEXT,
    base_url VARCHAR(500),
    is_default BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT NOW()
);
```

## AI 调用流程

```
用户操作 → API 请求 → AI Service
                         │
                    ┌────┴────┐
                    │ LiteLLM │ ← 统一接口
                    └────┬────┘
                         │
            ┌────────────┼────────────┐
            │            │            │
        OpenAI       Claude      Ollama (本地)
      GPT-4o/4o-mini  Sonnet     Qwen/Llama
```

### 关键 Prompt 模板

1. **剧情分析 Prompt**：输入章节内容 → 输出结构化摘要、人物列表、关键事件
2. **时间线提取 Prompt**：输入全文 → 输出时间-事件列表（JSON 格式）
3. **人物关系提取 Prompt**：输入章节 → 输出人物对和关系描述
4. **续写 Prompt**：输入原作风格分析 + 续写方向 → 输出同人文段落
5. **Embedding**：使用 text-embedding-3-small/large 生成章节向量
