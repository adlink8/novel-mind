## 技术栈（调研后更新）

| 层级 | 技术选型 | 版本 | 选型理由 |
|------|---------|------|---------|
| **前端框架** | Next.js (App Router) | 14+ | SSR + RSC，优秀的 SEO 和性能 |
| **前端语言** | TypeScript | 5.5+ | 类型安全，大型项目必备 |
| **UI 组件** | shadcn/ui + Tailwind CSS | - | 现代、可定制、生态成熟 |
| **时间线可视化** | D3.js + d3-timeline | 7+ | **调研推荐**：高定制性，支持缩放/平移/悬停详情/多角色并行线 |
| **人物关系图谱** | AntV G6 | 5+ | **调研推荐**：蚂蚁集团出品，中文文档友好，企业级力导向图 |
| **状态管理** | Zustand | 4+ | 轻量级，无 boilerplate |
| **后端框架** | Python FastAPI | 0.115+ | 异步高性能，AI 生态最佳 |
| **关系数据库** | PostgreSQL | 16 | 成熟稳定，支持 JSONB |
| **向量扩展** | pgvector | 0.7+ | 轻量向量检索，适合中小规模 |
| **向量数据库** | Chroma | - | **调研推荐**：Python 友好，快速上手；后续可迁移 Milvus |
| **图数据库** | Neo4j | 5+ | **调研推荐**：人物关系存储与查询的最佳方案，支持 Cypher |
| **AI 抽象层** | LiteLLM | 1.55+ | **调研推荐**：41.8K Stars，统一 100+ 模型 API |
| **NLP 引擎** | RexUniNLU | - | **调研推荐**：零样本中文 NER + 关系抽取，无需标注数据 |
| **Embedding** | BGE-large-zh-v1.5 | - | **调研推荐**：中文最优开源模型，可本地部署，1024 维 |
| **文风分析** | InkOS 文风指纹方案 | 自研 | **调研参考 InkOS**：提取句长分布、词频特征、叙事节奏 |
| **ORM** | SQLAlchemy 2.0 | 2.0+ | Python ORM 标准 |
| **数据库迁移** | Alembic | 1.14+ | SQLAlchemy 官方迁移工具 |
| **容器编排** | Docker Compose | - | 开发环境一键启动 |

### 技术栈变更说明（对比 v1.0）

| 变更项 | v1.0 | v2.0 | 变更原因 |
|--------|------|------|---------|
| 可视化 | ECharts | D3.js + AntV G6 | D3 定制性更强；G6 专为关系图谱设计 |
| Embedding | text-embedding-3-small | BGE-large-zh-v1.5 | 中文效果更优，开源可本地部署，降低成本 |
| 向量存储 | 仅 pgvector | pgvector + Chroma | Chroma 提供更丰富的检索能力（过滤、元数据），pgvector 保留为轻量方案 |
| 人物关系存储 | PostgreSQL | + Neo4j | 图数据库原生支持关系遍历、社区检测、最短路径等查询 |
| NER/关系抽取 | 纯 LLM | RexUniNLU + LLM 混合 | RexUniNLU 零样本抽取效率高、成本低，LLM 补充复杂场景 |
| 文风分析 | 无 | + InkOS 文风指纹 | 调研发现 InkOS 文风指纹提取是续写一致性的关键 |

## 核心数据模型

### PostgreSQL（关系数据）

```sql
-- 小说
CREATE TABLE novels (
    id UUID PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    author VARCHAR(255),
    description TEXT,
    genre VARCHAR(100),
    word_count INTEGER DEFAULT 0,
    chapter_count INTEGER DEFAULT 0,
    status VARCHAR(50) DEFAULT 'importing',  -- importing/chunking/embedding/analyzing/analyzed
    style_fingerprint JSONB,   -- 文风指纹（参考 InkOS：句长分布、词频、叙事节奏）
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 章节
CREATE TABLE chapters (
    id UUID PRIMARY KEY,
    novel_id UUID REFERENCES novels(id),
    chapter_number INTEGER,
    title VARCHAR(500),
    content TEXT NOT NULL,
    summary TEXT,
    word_count INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT NOW()
);

-- RAG 文本块（参考 Webnovel Writer 的 RAG 架构）
CREATE TABLE text_chunks (
    id UUID PRIMARY KEY,
    novel_id UUID REFERENCES novels(id),
    chapter_id UUID REFERENCES chapters(id),
    chunk_index INTEGER,          -- 章节内序号
    content TEXT NOT NULL,         -- 文本块内容（300-500 字）
    chunk_type VARCHAR(50),       -- scene/dialogue/description/narration
    metadata JSONB,               -- 角色名、地点等元数据
    embedding VECTOR(1024),       -- BGE 向量（1024 维）
    created_at TIMESTAMP DEFAULT NOW()
);
CREATE INDEX idx_chunks_novel ON text_chunks(novel_id);
CREATE INDEX idx_chunks_embedding ON text_chunks USING ivfflat (embedding vector_cosine_ops);

-- 人物（由 RexUniNLU + LLM 混合抽取）
CREATE TABLE characters (
    id UUID PRIMARY KEY,
    novel_id UUID REFERENCES novels(id),
    name VARCHAR(255) NOT NULL,
    aliases JSONB,                -- 别名列表
    description TEXT,
    personality JSONB,            -- 性格特征（多维向量）
    role VARCHAR(50),             -- protagonist/antagonist/supporting/minor
    first_appearance INTEGER,     -- 首次出场章节
    appearance_count INTEGER DEFAULT 0,  -- 总出场次数
    stats JSONB,                  -- 出场统计（每章出现次数）
    dialogue_style JSONB,         -- 对话风格特征
    created_at TIMESTAMP DEFAULT NOW()
);

-- 人物关系（同步到 Neo4j）
CREATE TABLE character_relations (
    id UUID PRIMARY KEY,
    novel_id UUID REFERENCES novels(id),
    source_character_id UUID REFERENCES characters(id),
    target_character_id UUID REFERENCES characters(id),
    relation_type VARCHAR(100),   -- family/friend/enemy/mentor/lover/rival/...
    description TEXT,
    strength FLOAT DEFAULT 0.5,   -- 关系强度 0-1
    evidence JSONB,               -- 证据（原文段落引用）
    chapter_range JSONB,          -- 关系存在的章节范围
    source VARCHAR(50),           -- 抽取来源：nlu_model/llm/manual
    created_at TIMESTAMP DEFAULT NOW()
);

-- 时间线事件
CREATE TABLE timeline_events (
    id UUID PRIMARY KEY,
    novel_id UUID REFERENCES novels(id),
    event_title VARCHAR(500) NOT NULL,
    description TEXT,
    event_time VARCHAR(255),      -- 小说内的时间描述
    chapter_id UUID REFERENCES chapters(id),
    event_order FLOAT,            -- 排序用
    event_type VARCHAR(100),      -- battle/meeting/discovery/death/revelation/...
    importance VARCHAR(20),       -- high/medium/low
    characters_involved UUID[],   -- 涉及的人物 ID
    causal_links UUID[],          -- 因果关联事件 ID
    source VARCHAR(50),           -- 抽取来源
    created_at TIMESTAMP DEFAULT NOW()
);

-- 同人文
CREATE TABLE fanfictions (
    id UUID PRIMARY KEY,
    novel_id UUID REFERENCES novels(id),
    title VARCHAR(500) NOT NULL,
    description TEXT,
    style_config JSONB,           -- 风格参数（文风强度、视角、情感基调）
    branch_point JSONB,           -- 分支点（时间线事件 ID + 分支描述）
    status VARCHAR(50) DEFAULT 'draft',
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 同人文章节
CREATE TABLE fanfiction_chapters (
    id UUID PRIMARY KEY,
    fanfiction_id UUID REFERENCES fanfictions(id),
    chapter_number INTEGER,
    title VARCHAR(500),
    content TEXT,
    ai_generated BOOLEAN DEFAULT FALSE,
    model_used VARCHAR(100),
    style_score FLOAT,            -- 风格一致性评分（对比原作指纹）
    rag_context JSONB,            -- RAG 检索到的参考段落
    created_at TIMESTAMP DEFAULT NOW()
);

-- AI 模型配置
CREATE TABLE ai_configs (
    id UUID PRIMARY KEY,
    user_id VARCHAR(255),
    provider VARCHAR(50),         -- openai/anthropic/ollama/custom
    model_name VARCHAR(100),
    api_key_encrypted TEXT,
    base_url VARCHAR(500),
    is_default BOOLEAN DEFAULT FALSE,
    routing_preference VARCHAR(50) DEFAULT 'balanced',  -- quality/balanced/budget
    created_at TIMESTAMP DEFAULT NOW()
);

-- AI 调用日志（成本追踪）
CREATE TABLE ai_usage_logs (
    id UUID PRIMARY KEY,
    model_name VARCHAR(100),
    task_type VARCHAR(50),        -- analysis/embedding/fanfiction/nlp
    input_tokens INTEGER,
    output_tokens INTEGER,
    cost_usd FLOAT,
    latency_ms INTEGER,
    created_at TIMESTAMP DEFAULT NOW()
);
```

### Neo4j（人物关系图谱）

```cypher
// 节点标签
// :Character { id, name, novel_id, role, appearance_count }
// :Novel { id, title }

// 关系类型
// [:RELATED_TO { type, strength, description, chapter_range }]
// [:APPEARS_IN { chapter_number }]
// [:ALLIED_WITH { strength, from_chapter, to_chapter }]
// [:OPPOSED_TO { strength, from_chapter, to_chapter }]

// 示例查询：获取某小说的核心人物关系子图
MATCH (c:Character {novel_id: $novel_id})-[r:RELATED_TO]->(c2:Character)
WHERE r.strength > 0.3
RETURN c, r, c2

// 示例查询：查找某角色的所有关联角色（2度以内）
MATCH path = (c:Character {name: $name, novel_id: $novel_id})-[:RELATED_TO*1..2]-(other)
RETURN path

// 示例查询：社区检测（发现阵营/派系）
CALL gds.louvain.stream('character-graph')
YIELD nodeId, communityId
RETURN gds.util.asNode(nodeId).name AS name, communityId
ORDER BY communityId
```

## AI 智能路由策略

参考调研中 LiteLLM + 智能路由的方案，按任务类型自动选择最优模型：

```
                    ┌──────────────────┐
                    │   AI Router      │
                    │  (ai_router.py)  │
                    └────────┬─────────┘
                             │
          ┌──────────────────┼──────────────────┐
          │                  │                  │
    ┌─────┴─────┐    ┌──────┴──────┐    ┌──────┴──────┐
    │ 高质量任务 │    │ 中等任务    │    │ 轻量任务    │
    │           │    │             │    │             │
    │ Claude    │    │ Claude      │    │ GPT-4o-mini │
    │ Opus/     │    │ Sonnet/     │    │ Qwen-72B/   │
    │ GPT-4o    │    │ GPT-4o-mini │    │ 本地模型    │
    └───────────┘    └─────────────┘    └─────────────┘
```

| 任务类型 | 推荐模型 | 原因 |
|----------|---------|------|
| 剧情深度分析 | Claude Opus / GPT-4o | 需要深度理解和推理能力 |
| 叙事结构分析 | Claude Opus / GPT-4o | 需要叙事学知识 |
| 人物/关系抽取 | RexUniNLU → Claude Sonnet 兜底 | NLU 零样本抽取成本低，复杂场景用 LLM |
| 时间线事件提取 | Claude Sonnet / GPT-4o-mini | 结构化提取，中等复杂度 |
| 续写/同人创作 | Claude Opus / GPT-4o | 需要创意能力和风格模仿 |
| 文风指纹分析 | 本地计算 + Claude Sonnet | 统计特征本地算，风格判断用 LLM |
| Embedding | BGE-large-zh-v1.5（本地） | 中文最优，零 API 成本 |
| 简单摘要/格式化 | GPT-4o-mini / Qwen | 成本低，速度快 |

用户可在设置中切换三档预设：**质量优先** / **均衡** / **经济模式**。

                            │
                            ▼
                   ┌────────────────┐
                   │ 三级分块       │
                   │ 章 > 场景 > 段 │
                   │ (300-500字/块) │
                   └────────┬───────┘
                            │
                            ▼
                   ┌────────────────┐
                   │ BGE Embedding  │
                   │ (1024 维向量)  │
                   └────────┬───────┘
                            │

参考调研中 RexUniNLU 的零样本抽取能力，结合 LLM 处理复杂场景：

小说文本
    │
    ├──▶ RexUniNLU（本地，零样本）

    │    └── 叙事结构分析
    │
    ▼
┌──────────────────────────────────┐

│  (优先级：人工 > LLM > NLU)     │

    "sentence_length": {

    "vocabulary": {
        "unique_ratio": 0.42,   # 词汇丰富度
        "top_words": [...],     # 高频词 Top 50

    "narrative": {
        "dialogue_ratio": 0.35, # 对话占比
        "description_ratio": 0.25, # 描写占比

    },

        "paragraph_length_mean": 120, # 平均段落长度

## 风险缓解方案

| 风险 | 影响 | 缓解策略 |
|------|------|---------|
| LLM 超长文本局限 | >100 万字理解不完整 | RAG 分块检索 + 三级分块策略 + 章节摘要链 |
| 中文 NLP 质量 | 关系/时间线抽取不准确 | RexUniNLU + LLM 混合抽取 + 人工修正机制 |
| AI 调用成本高 | 多模型 + 长文本 Token 费用高 | 智能路由分层 + BGE 本地 Embedding + 缓存 + 成本追踪 |
| 续写风格不一致 | 同人文"AI味"重 | 文风指纹约束 + RAG 上下文注入 + 多 Agent 审计 |
| 版权问题 | 导入受版权保护小说 | 明确使用条款 + 仅限个人分析 + 不存储原文（可选） |
| Neo4j 运维复杂度 | 图数据库增加部署成本 | 初期可仅用 PostgreSQL，达到关系规模阈值后引入 Neo4j |
