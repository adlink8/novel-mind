## Phase 2：小说导入 + RAG 索引（Sprint 3-4，预计 2-3 周）

**目标**：用户可以导入小说、在线阅读，并建立 RAG 语义索引

### 任务

**导入管线**
- [ ] 2.1 TXT 文件上传（前端拖拽上传 + 后端文件处理 + 进度反馈）
- [ ] 2.2 文本清洗（chardet 编码检测 → UTF-8 规范化 → 去除噪音字符）
- [ ] 2.3 章节自动分割（正则匹配：第X章/第X回/Chapter X 等模式）
- [ ] 2.4 小说元信息提取（字数统计、章节数、预估作者/类型）

**RAG 索引（参考 Webnovel Writer 架构）**
- [ ] 2.5 三级文本分块（章节 → 场景 → 段落，每块 300-500 字）
- [ ] 2.6 分块元数据标注（chapter_number、characters、location、chunk_type）
- [ ] 2.7 BGE-large-zh-v1.5 Embedding 服务（本地部署，生成 1024 维向量）
- [ ] 2.8 向量写入 Chroma（含元数据过滤条件）
- [ ] 2.9 轻量向量同步到 pgvector（备用检索路径）

**阅读与搜索**
- [ ] 2.10 小说列表页（卡片式展示、搜索、排序、状态标签）
- [ ] 2.11 小说阅读页（章节导航、阅读设置、目录、阅读进度记忆）
- [ ] 2.12 混合语义搜索（向量相似度 + BM25 关键词 + 元数据过滤）
  - 用户可输入自然语言查询："林黛玉第一次生病的场景"
  - 返回匹配的文本块，高亮显示并跳转到原文

**交付物**：导入 TXT → 自动分块索引 → 在线阅读 → 全文语义搜索

**借鉴来源**：
- Webnovel Writer（22K Stars）：RAG + 向量检索解决长篇"遗忘"问题
- BGE Embedding：中文最优开源方案

## Phase 3：AI 剧情分析 + NLP 管线（Sprint 5-7，预计 3 周）

**目标**：AI 深度分析小说内容，自动识别人物、关系、伏笔、叙事结构

### 任务

**AI 基础设施**
- [ ] 3.1 LiteLLM 多模型集成层（OpenAI / Claude / Ollama / 自定义统一接口）
- [ ] 3.2 AI 智能路由（ai_router.py）
  - 按任务类型分配模型：深度分析→高质量 / 抽取→中等 / 摘要→轻量
  - 三档预设：质量优先 / 均衡 / 经济模式
- [ ] 3.3 AI 模型配置 UI（添加/切换/测试连接、API Key 管理）
- [ ] 3.4 AI 调用成本追踪（token 用量 + 费用统计 + 预算限制）

**NLP 管线（RexUniNLU + LLM 混合）**
- [ ] 3.5 RexUniNLU 本地 NLP 服务部署（Docker + GPU 可选）
- [ ] 3.6 NER 抽取（人物名称、别名、地点自动识别）
- [ ] 3.7 关系三元组抽取（人物A, 关系类型, 人物B）
- [ ] 3.8 事件检测（关键动作/事件自动标注）
- [ ] 3.9 LLM 补充抽取（复杂关系判断、性格分析、伏笔检测）
- [ ] 3.10 结果合并与冲突消解（人工 > LLM > NLU 优先级）

**分析引擎**
- [ ] 3.11 章节摘要生成（每章 200-500 字结构化摘要）
- [ ] 3.12 全书剧情摘要（分卷/分弧总结 + 主线/支线梳理）
- [ ] 3.13 叙事结构分析（参考 Narrative Theory-Driven LLM 论文）
  - 叙事视角检测、角色功能识别、情节弧线标注
- [ ] 3.14 伏笔/悬念/转折点标注（跨章节因果关联）
- [ ] 3.15 人物性格分析（基于对话和行为描写）

- [ ] 3.18 流式输出支持（SSE，分析过程实时展示进度）
- [ ] 3.19 分析结果人工修正（修正 AI 识别错误，反馈到模型）

**交付物**：导入小说后一键生成完整的剧情分析报告（摘要 + 人物 + 伏笔 + 叙事结构 + 情感曲线）

**借鉴来源**：
- RexUniNLU：零样本中文 NER + 关系抽取

## MVP 降级策略

如果时间紧张，以下模块可以降级或延后，不影响核心体验：

| 模块 | 完整方案 | MVP 降级方案 | 影响 |
|------|---------|-------------|------|
| Neo4j | 独立图数据库 | 仅 PostgreSQL 存储关系 | 丧失复杂图查询和社区检测 |
| Chroma | 独立向量库 | 仅 pgvector | 检索功能较弱但可用 |
| RexUniNLU | 本地 NLP 服务 | 纯 LLM 抽取 | 成本增加、速度变慢 |
| BGE 本地部署 | 本地 Embedding | 调用 OpenAI Embedding API | 增加 API 成本 |
| 多 Agent 审计 | 审计员+修订员 | 单次 LLM 自检 | 续写质量可能下降 |
| 文风指纹 | 完整统计分析 | 简单 Prompt 描述风格 | 续写一致性减弱 |

MVP 最快路径：**仅 PostgreSQL + pgvector + LiteLLM + LLM 全流程**，可在 10-12 周内上线。
