# NovelMind 项目路线图

## Phase 1：基础搭建（Sprint 1-2，预计 2-3 周）

**目标**：跑通前后端基础框架 + 数据库 + AI 接入

### 任务
- [ ] 1.1 初始化 Next.js 项目（App Router + TypeScript + Tailwind + shadcn/ui）
- [ ] 1.2 初始化 FastAPI 项目（项目结构 + 配置管理 + 日志）
- [ ] 1.3 PostgreSQL + pgvector 数据库搭建（Docker）
- [ ] 1.4 设计并创建数据表（SQLAlchemy ORM + Alembic 迁移）
- [ ] 1.5 搭建全局 UI 布局（侧边栏导航 + 顶栏 + 响应式）
- [ ] 1.6 Docker Compose 开发环境编排
- [ ] 1.7 基础 API 框架（错误处理、CORS、中间件）

**交付物**：可运行的前后端骨架，访问首页能看到布局框架

---

## Phase 2：小说导入与阅读（Sprint 3-4，预计 2-3 周）

**目标**：用户可以导入小说并在线阅读

### 任务
- [ ] 2.1 TXT 文件上传（前端上传组件 + 后端文件处理）
- [ ] 2.2 文本清洗（编码检测、格式规范化）
- [ ] 2.3 章节自动分割（基于标题模式识别：第X章、第X回等）
- [ ] 2.4 小说元信息提取（字数统计、章节数、预估作者）
- [ ] 2.5 小说列表页（卡片式展示、搜索、排序）
- [ ] 2.6 小说阅读页（章节导航、阅读设置、目录）
- [ ] 2.7 内容向量化（章节 Embedding → pgvector）
- [ ] 2.8 语义搜索功能（基于向量的剧情搜索）

**交付物**：可以导入 TXT 小说，在线阅读，全文语义搜索

---

## Phase 3：AI 剧情分析（Sprint 5-6，预计 2-3 周）

**目标**：AI 深度分析小说内容

### 任务
- [ ] 3.1 LiteLLM 多模型集成层（OpenAI/Claude/Ollama 统一接口）
- [ ] 3.2 AI 模型配置 UI（添加/切换/测试模型连接）
- [ ] 3.3 章节摘要生成（每章自动生成 200-500 字摘要）
- [ ] 3.4 全书剧情摘要（分卷/分弧总结）
- [ ] 3.5 人物自动识别（NER + LLM 辅助）
- [ ] 3.6 人物性格分析（基于对话和行为描写）
- [ ] 3.7 伏笔/悬念/转折点标注
- [ ] 3.8 剧情分析结果展示页（结构化呈现）
- [ ] 3.9 流式输出支持（SSE，分析过程实时展示）

**交付物**：导入小说后一键生成完整的剧情分析报告

---

## Phase 4：时间线可视化（Sprint 7-8，预计 2 周）

**目标**：将小说事件以时间线形式可视化呈现

### 任务
- [ ] 4.1 AI 时间线事件提取（从全文提取时间标记和事件）
- [ ] 4.2 时间线数据模型与 API
- [ ] 4.3 ECharts 时间线组件（横向/纵向可切换）
- [ ] 4.4 事件详情弹窗（点击查看完整描述）
- [ ] 4.5 多视角时间线（按角色筛选）
- [ ] 4.6 时间线手动编辑（添加/修改/删除/拖拽排序）
- [ ] 4.7 事件因果链可视化（连线展示因果关系）

**交付物**：交互式时间线页面，支持筛选、编辑、多视角

---

## Phase 5：人物关系图谱（Sprint 9-10，预计 2 周）

**目标**：构建可交互的人物关系网络

### 任务
- [ ] 5.1 AI 人物关系提取（关系类型、强度、章节范围）
- [ ] 5.2 关系数据 API（增删改查 + 图查询）
- [ ] 5.3 ECharts/D3 关系图谱组件（力导向图）
- [ ] 5.4 关系筛选（按类型、按强度、按章节范围）
- [ ] 5.5 关系演变动画（拖动时间轴看关系变化）
- [ ] 5.6 人物卡片页面（详细信息、出场统计、经典台词）
- [ ] 5.7 人物手动编辑（修正 AI 识别错误）

**交付物**：可交互的人物关系图谱 + 人物详情卡片

---

## Phase 6：AI 续写/同人文（Sprint 11-13，预计 3 周）

**目标**：基于原作风格续写同人小说

### 任务
- [ ] 6.1 原作风格分析引擎（文风、用词、句式、叙事节奏）
- [ ] 6.2 续写参数配置 UI（视角、风格强度、续写方向）
- [ ] 6.3 续写生成引擎（流式输出 + 迭代修改）
- [ ] 6.4 分支点选择（从时间线某个事件开始分支）
- [ ] 6.5 富文本编辑器（Markdown / 所见即所得）
- [ ] 6.6 风格一致性评分（对比原作风格）
- [ ] 6.7 续写章节管理（增删改、排序）
- [ ] 6.8 续写历史与版本管理

**交付物**：完整的同人文创作工作流，从选分支到续写到编辑

---

## Phase 7：打磨与部署（Sprint 14-15，预计 2 周）

**目标**：上线可用版本

### 任务
- [ ] 7.1 响应式布局全面适配（移动端/平板/桌面）
- [ ] 7.2 性能优化（API 缓存、图片优化、懒加载）
- [ ] 7.3 用户认证系统（NextAuth.js）
- [ ] 7.4 全局错误处理与友好提示
- [ ] 7.5 日志系统与基础监控
- [ ] 7.6 Docker 生产环境配置
- [ ] 7.7 部署文档与一键部署脚本
- [ ] 7.8 用户引导流程（新手教程）

**交付物**：可部署上线的生产版本
