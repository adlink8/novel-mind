## Phase 概览

### Phase 1：基础搭建 ✅ 已完成
前后端框架、数据存储（PostgreSQL + Chroma + Neo4j）、API 契约、CI。

### Phase 2：小说导入 + RAG 索引（当前）
TXT 导入 → 自动分块索引 → 在线阅读 → 语义搜索。当前核心能力已验证，剩余混合搜索 + 持久化导入。

### Phase 3：AI 剧情分析 + NLP 管线（规划）
LiteLLM 多模型路由、NER 抽取、人物关系、章节摘要、叙事结构分析、情感曲线。

### Phase 4：时间线可视化（规划）
D3.js 交互式时间线：事件提取、多视角、因果链、拖拽编辑。

### Phase 5：人物关系图谱（规划）
Neo4j + AntV G6 力导向图：社区检测、关系演变、人物卡片。

### Phase 6：AI 续写/同人文（规划）
文风指纹提取 → RAG 上下文注入 → 多 Agent 审计 → 富文本编辑 → 多格式导出。

### Phase 7：打磨与部署（规划）
响应式适配、性能优化、用户认证（NextAuth.js）、生产部署、成本监控。

## MVP 降级策略

| 模块 | 完整方案 | MVP 降级 | 影响 |
|------|---------|---------|------|
| Neo4j | 独立图数据库 | PostgreSQL 存储关系 | 丧失复杂图查询 |
| Chroma | 独立向量库 | 仅 pgvector | 检索功能减弱 |
| RexUniNLU | 本地 NLP | 纯 LLM 抽取 | 成本增加 |
| BGE 本地部署 | 本地 Embedding | OpenAI API | 增加 API 成本 |
| 多 Agent 审计 | 审计员+修订员 | 单次 LLM 自检 | 续写质量下降 |

MVP 最快路径：**PostgreSQL + pgvector + LiteLLM + LLM**，可在 10-12 周内上线。
